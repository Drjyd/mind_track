# OneNote + Obsidian + 思维导图知识处理工作台

这是一个仅依赖 Python 标准库的本地工具，用于把临时信息逐步转化为可追溯、可验证、可复用的研究知识。

它不会把未提供的信息自动包装成“结论”。每条记录都可以标记为事实、假设、推论、经验或未知问题，并补充思维方式、关系类型、工具层、来源、证据、置信度和下一步验证。

## 启动

在本目录打开 PowerShell：

```powershell
python .\idea_framework.py ui
```

也可以直接运行 `python .\idea_framework.py`，在快速命令行中输入一条记录。

## 现在更适合日常使用

- 首页概览显示知识记录、主题、待验证与收藏数量。
- 历史记录支持关键词、知识类型、置信度和收藏组合筛选，双击即可编辑。
- 输入内容会自动暂存；应用异常退出或误关闭后，下次启动会恢复未保存草稿。
- 删除后可在当前会话撤销；清空输入前会二次确认。
- 知识报告支持拖选后删除单行内容，也可按章节批量删除；管理窗口提供“全选所有”和 `Ctrl+A`，所有删除均可恢复，且不会删除底层知识记录。
- 编辑窗口支持滚动，小屏幕下仍可访问全部字段和保存按钮。
- 导出与备份失败会显示明确原因，不会无提示中断。

### 键盘快捷键

- `Ctrl+N`：定位到快速输入
- `Ctrl+Enter`：保存当前记录
- `Ctrl+F`：定位到历史搜索
- `Esc`：清除搜索与筛选
- `F5`：刷新报告、导图与主题
- 编辑窗口中 `Ctrl+S` 保存、`Esc` 取消

## 推荐工作流

1. 在“快速输入”保留 OneNote 式原始信息。
2. 在“结构化输入”补充机制、应用场景、验证标准和失败风险。
3. 在“标注与证据”选择知识状态、六种思维方式、关系类型和工具层，并登记来源、证据、置信度、链接与下一步。
4. 保存后检查“15段知识报告”和“关系导图”，确认假设没有被写成事实。
5. 使用“导出知识包”，得到完整报告、OneNote 清单、Obsidian 原子笔记与 MOC、Mermaid 和可浏览网页。

## 六种思维方式

- 第一性原理
- 系统性思维
- 过程思维
- 反向思维
- 默会知识
- 高杠杆思考

导图节点使用以下真实关系标签：包含、因果、依赖、对比、时序、反馈、证据、风险、行动。

## 输出文件

- `ideas.jsonl`：原始知识条目，兼容旧版想法记录。
- `framework.md`：按提示词生成的 15 段可检查报告。
- `idea_framework_meta.json`：人工主题名、固定归类与收藏状态。
- `framework_visual.mmd`：关系型 Mermaid 思维导图。
- `framework_visual.html`：浏览器可打开的知识框架。
- `onenote_capture.md`：待在 OneNote 保留或复盘的原始记录。
- `obsidian_export/`：原子笔记和主题 MOC，可复制到 Obsidian 库。

## 命令行示例

```powershell
python .\idea_framework.py add `
  --title "实验温漂来自探测器增益" `
  --summary "连续工作后信号基线逐步抬升" `
  --knowledge-type "假设" `
  --thinking-mode "第一性原理" `
  --relation-type "因果" `
  --tool-layer "OneNote原始捕捉层" `
  --source "2026-07-14 实验记录" `
  --confidence "待验证" `
  --next-action "恒温条件下重复三轮并比较基线"

python .\idea_framework.py framework
python .\idea_framework.py export
```

## 设计约束

- 导图层级不超过五层。
- 每个主要分支最多显示七条记录，避免信息过载；完整内容仍保留在报告和笔记中。
- 低置信度、假设、推论和未知问题会明确标记“待验证”。
- Obsidian 导出只新增或覆盖同名生成文件，不删除用户已有文件。
